📋 任务 1：详细步骤

启用虚拟化技术

Hyper-V 用户：控制面板 → 程序 → 启用或关闭 Windows 功能 → 勾选 Hyper-V

VirtualBox 用户：下载并安装 VirtualBox（已安装可跳过）

创建新虚拟机

打开虚拟化软件，点击"新建"或"创建虚拟机"

命名：Ubuntu-Lab

类型：Linux，版本：Ubuntu (64-bit)

分配资源（重要！）

  • CPU：2-4 核心（不超过物理核心的 50%）
  • 内存：4GB（4096 MB）
  • 硬盘：40GB（动态分配）

挂载 ISO 镜像

在虚拟机设置中，找到"存储"或"光驱"选项

选择你下载的 Ubuntu ISO 文件

启动并安装系统

启动虚拟机，按照安装向导完成 Ubuntu 安装

语言选择：中文（简体）或 English

安装类型：清除整个磁盘并安装 Ubuntu

⚙️ 虚拟机资源分配原则

❌ 错误做法

  • 分配所有 CPU 核心
  • 分配 90% 的内存
  • 固定分配硬盘空间
  • 不考虑主机性能

结果：主机卡顿，虚拟机也慢

✅ 正确做法

  • CPU：不超过物理核心的 50%
  • 内存：不超过物理内存的 50%
  • 硬盘：使用动态分配
  • 根据实际需求调整

结果：主机和虚拟机都流畅

💡

经验法则：8GB 物理内存的电脑，给虚拟机分配 4GB；16GB 的分配 6-8GB

🌐 网络模式实验

🔬 实验：观察网络模式的差异

安装完成后，我们来做一个有趣的实验

NAT 模式测试

ip a # 查看 IP 地址
ping baidu.com # 测试外网连接
ip route # 查看路由表

记录下 IP 地址（通常是 10.0.x.x 或 192.168.x.x）

切换到桥接模式

关闭虚拟机 → 虚拟机设置 → 网络 → 选择"桥接模式"

重新启动虚拟机

桥接模式测试

ip a # 再次查看 IP
ping baidu.com # 测试连接

观察：IP 地址变了吗？和你的主机在同一网段吗？

🔍 网络模式深度对比

NAT 模式

🏠
  • 虚拟机可以上网
  • 主机可以访问虚拟机
  • 外部网络无法访问虚拟机
  • IP：10.0.x.x 或 192.168.x.x

✅ 推荐日常使用

桥接模式

🌉
  • 虚拟机获得独立 IP
  • 像局域网中的真实电脑
  • 外部可以直接访问
  • IP：与主机同网段

✅ 适合搭建服务器

仅主机模式

🔒
  • 只能与主机通信
  • 完全隔离外部网络
  • 无法上网
  • IP：192.168.56.x

✅ 适合安全测试

🎯 任务 3：实践练习

向 AI 提问

分析 AI 的回答

AI 通常会建议：

  • sudo apt update - 更新软件包列表
  • sudo apt upgrade - 升级已安装的软件

💡 不要盲目复制，先理解每个命令的作用

在终端中执行

sudo apt update
# 输入密码（输入时不显示，这是正常的）
# 观察输出，看到 "Reading package lists... Done" 表示成功

继续向 AI 学习

如果遇到不理解的地方，继续提问：

"sudo 是什么意思？为什么需要它？"
"apt 和 apt-get 有什么区别？"

    • ❓ 实验一：常见问题

    Q1: 虚拟机启动很慢怎么办？

    A: 检查是否分配了过多资源，或者主机内存不足。建议关闭其他占用资源的程序。

    Q2: 无法启用 Hyper-V？

    A: 确保你的 Windows 是 Pro 或 Enterprise 版本，Home 版不支持。或者使用 VirtualBox 替代。

    Q3: 虚拟机无法上网？

    A: 检查网络模式是否设置为 NAT，或者尝试重启虚拟机的网络服务：sudo systemctl restart NetworkManager

    Q4: 忘记了 Ubuntu 的密码？

    A: 恢复到安装后的快照，或者通过 GRUB 引导进入单用户模式重置密码（可以问 AI 具体步骤）。

    📍 绝对路径探索

    实验：使用绝对路径导航
    student@ubuntu:~$
    pwd
    /home/student
    student@ubuntu:~$
    cd /etc
    student@ubuntu:/etc$
    pwd
    /etc
    student@ubuntu:/etc$
    ls -la | head -10
    total 1234
    drwxr-xr-x 130 root root 12288 Mar 12 10:30 .
    drwxr-xr-x 20 root root 4096 Mar 10 15:20 ..
    -rw-r--r-- 1 root root 3028 Mar 1 08:15 adduser.conf
    drwxr-xr-x 2 root root 4096 Mar 10 15:25 apt
    💡

    绝对路径总是从 / 开始，无论你在哪里，路径都一样

    🔄 相对路径探索

    实验：使用相对路径导航
    student@ubuntu:/etc$
    cd ../var/log
    student@ubuntu:/var/log$
    pwd
    /var/log
    student@ubuntu:/var/log$
    cd ../..
    student@ubuntu:/$
    pwd
    /
    student@ubuntu:/$
    cd ~
    student@ubuntu:~$
    pwd
    /home/student
    💡

    .. 表示上级目录，~ 表示主目录，. 表示当前目录

    👀 解读 ls -la 输出

    drwxr-xr-x 2 student student 4096 Mar 12 10:30 Documents
    -rw-r--r-- 1 student student 1024 Mar 11 15:20 file.txt
    -rwxr-xr-x 1 student student 8192 Mar 10 09:15 script.sh

    第一列：类型和权限

    • d = 目录
    • - = 普通文件
    • rwx = 读、写、执行
    • 三组权限：所有者、组、其他人

    其他信息

    • 链接数
    • 所有者和组
    • 文件大小（字节）
    • 修改时间
    • 文件名

    🔍 发现隐藏文件

    实验：查看隐藏文件
    student@ubuntu:~$
    ls
    Desktop Documents Downloads Music Pictures Videos
    student@ubuntu:~$
    ls -a
    . .. .bash_history .bashrc .profile .cache Desktop Documents
    student@ubuntu:~$
    ls -la | grep "^\."
    drwxr-xr-x 5 student student 4096 Mar 12 10:30 .
    drwxr-xr-x 3 root root 4096 Mar 10 15:20 ..
    -rw------- 1 student student 2048 Mar 12 09:15 .bash_history
    -rw-r--r-- 1 student student 220 Mar 10 15:20 .bash_logout
    💡

    Linux 中，以 . 开头的文件是隐藏文件，通常是配置文件

    📁 创建项目目录结构

    回到主目录

    cd ~
    pwd # 确认在 /home/student

    创建多层目录（一次性）

    mkdir -p myproject/src myproject/logs myproject/docs
    # -p 参数会自动创建父目录

    验证目录结构

    tree myproject
    # 如果没有 tree 命令，使用：
    ls -R myproject

    输出应该类似：

    myproject/

    ├── logs
    └── src

    ✨ 批量创建文件

    实验：使用 touch 批量创建
    student@ubuntu:~$
    cd myproject/src
    student@ubuntu:~/myproject/src$
    touch main.c utils.c config.h
    ls -l
    -rw-r--r-- 1 student student 0 Mar 12 10:35 config.h
    -rw-r--r-- 1 student student 0 Mar 12 10:35 main.c
    -rw-r--r-- 1 student student 0 Mar 12 10:35 utils.c
    student@ubuntu:~/myproject/src$
    cd ../docs
    touch README.md INSTALL.md API.md
    💡

    touch 可以一次创建多个文件，用空格分隔文件名

    🚀 批量创建的高级技巧

    使用花括号展开
    student@ubuntu:~$
    mkdir -p project/{src,bin,lib,include}
    # 一次创建 4 个目录
    student@ubuntu:~$
    touch file{1..10}.txt
    # 创建 file1.txt 到 file10.txt
    student@ubuntu:~$
    touch report.{txt,pdf,docx}
    # 创建 report.txt, report.pdf, report.docx
    student@ubuntu:~$
    mkdir -p year_{2020..2024}/month_{01..12}
    # 创建 5 年 × 12 月 = 60 个目录
    💡

    花括号展开是 Shell 的强大功能，可以大大提高效率

    📋 复制文件实验

    cp 命令实践
    student@ubuntu:~$
    cd myproject/src
    cp main.c main_backup.c
    ls -l
    -rw-r--r-- 1 student student 0 Mar 12 10:40 main.c
    -rw-r--r-- 1 student student 0 Mar 12 10:41 main_backup.c
    student@ubuntu:~/myproject/src$
    cp -r ../docs ../docs_backup
    # 复制整个目录需要 -r 参数
    student@ubuntu:~/myproject/src$
    cp -i main.c main_backup.c
    cp: overwrite 'main_backup.c'? y
    # -i 参数会在覆盖前询问

    🚚 移动和重命名实验

    mv 命令实践

    # 重命名文件
    student@ubuntu:~/myproject/src$
    mv tools.c ~/
    # 移动到主目录
    student@ubuntu:~/myproject/src$
    mv ~/tools.c ~/utilities.c
    # 移动并重命名
    student@ubuntu:~/myproject/src$
    cd ~ && mv myproject my_awesome_project
    # 重命名目录
    💡

    mv 不需要 -r 参数就可以移动目录，因为它只是改变路径，不复制内容

    ⚖️ cp vs mv 对比

    cp（复制）

    • 创建文件的副本
    • 原文件保留
    • 复制目录需要 -r
    • 占用额外空间
    • 常用参数：-r、-i、-p
    cp file1 file2
    cp -r dir1 dir2

    mv（移动/重命名）

    • 移动或重命名
    • 原文件消失
    • 不需要 -r 参数
    • 不占用额外空间
    • 常用参数：-i
    mv file1 file2
    mv dir1 dir2

    危险的 rm 命令

    ⏱️ 15 分钟 高级

    理解 rm 的危险性，学会安全删除文件

    ⚠️

    重要警告

    Linux 没有回收站！rm 删除的文件无法恢复！

    在执行此任务前，请确保已创建虚拟机快照！

    🛡️ 安全删除练习

    创建测试文件

    cd ~
    mkdir test_delete
    cd test_delete
    touch file{1..5}.txt
    ls

    使用 -i 参数安全删除

    rm -i file1.txt
    rm: remove regular empty file 'file1.txt'? y
    # 输入 y 确认删除，n 取消

    查看 man 手册

    man rm
    # 按 / 搜索 "interactive"
    # 找到 -i 参数的详细说明
    # 按 q 退出

    删除整个目录

    cd ~
    rm -ri test_delete
    # -r 递归删除，-i 逐一确认

    ⚠️ rm 命令危险等级

    ✅ 安全

    rm -i file.txt
    rm -ri directory

    删除前逐一确认

    ⚠️ 小心

    rm file.txt
    rm -r directory

    直接删除，无法恢复

    🚨 危险

    rm -rf directory
    rm -rf *

    强制递归删除，极度危险

    ☠️ 致命

    rm -rf /
    sudo rm -rf /*

    删除整个系统！永远不要执行！

    🛡️ rm 使用最佳实践

    1️⃣

    删除前先查看

    ls directory
    # 确认内容后再删除
    rm -r directory
    2️⃣

    使用 -i 参数

    rm -i file.txt
    rm -ri directory

    养成确认的习惯

    3️⃣

    避免使用通配符

    # 危险！
    rm -rf *
    # 安全
    rm -i file1 file2
    4️⃣

    重要文件先备份

    cp important.txt backup/
    # 确认备份后再删除
    rm important.txt

    • ❓ 实验二：常见问题

    Q1: mkdir -p 和 mkdir 有什么区别？

    A: mkdir -p 会自动创建父目录，而 mkdir 如果父目录不存在会报错。例如：mkdir -p a/b/c 会创建 a、b、c 三个目录。

    Q2: 为什么 tail -f 看不到日志更新？

    A: 可能是权限问题，尝试使用 sudo tail -f /var/log/syslog。或者日志文件路径不对，不同发行版日志位置可能不同。

    Q3: 误删了文件怎么办？

    A: 如果在虚拟机中，恢复快照。如果是真实系统，Linux 没有回收站，只能尝试数据恢复工具（成功率低）。所以删除前一定要确认！

    Q4: ls -la 中的 . 和 .. 是什么？

    A: . 代表当前目录，.. 代表上级目录。它们是特殊的目录引用，存在于每个目录中。
