🔍 虚拟机的两种类型

Type 1 - 裸机虚拟化

🏗️
  • Hyper-V
  • VMware ESXi
  • KVM
  • 直接运行在硬件上
  • 性能更好
  • 企业级应用

Type 2 - 托管虚拟化

📦
  • VirtualBox
  • VMware Workstation
  • Parallels Desktop
  • 运行在操作系统上
  • 易于安装使用
  • 个人学习首选

💎 Hyper-V 深入了解

  • Windows 内置的虚拟化平台（Pro/Enterprise 版本）
  • Type 1 虚拟化，性能接近原生
  • 与 Windows 系统深度集成
  • 支持动态内存分配
  • 完全免费使用
  • 缺点：仅限 Windows 系统，不支持 Home 版

⚙️ 虚拟机资源分配

  • CPU：分配处理器核心数（建议 2-4 核）
  • 内存：分配 RAM 大小（建议 4-8 GB）
  • 硬盘：虚拟磁盘空间（建议 40-60 GB）
  • 网络：NAT、桥接、仅主机模式
  • 💡 原则：不要分配超过物理机 50% 的资源

🐳 容器 vs 虚拟机

虚拟机

  • 完整的操作系统
  • 资源占用大（GB 级别）
  • 启动慢（分钟级）
  • 隔离性强
  • 适合运行不同系统

容器（Docker）

  • 共享主机内核
  • 资源占用小（MB 级别）
  • 启动快（秒级）
  • 轻量级隔离
  • 适合微服务架构

📁 Linux 文件系统

一切皆文件的哲学

💡 Linux 内核统一管理所有文件

🌟 一切皆文件

  • 普通文件：文档、图片、程序
  • 目录也是文件（特殊的文件）
  • 硬件设备也是文件（/dev/sda 是硬盘）
  • 进程信息也是文件（/proc 目录）
  • 网络连接也是文件（/proc/net）
  • 这种设计让一切都可以用统一的方式操作

🌳 Linux 目录树

/ (根目录 - 一切的起点)
├── home/ (用户主目录)
│ └── student/ (你的个人空间)
├── etc/ (配置文件)
├── usr/ (程序和库)
├── var/ (变化的数据)
├── tmp/ (临时文件)
├── dev/ (设备文件)
├── bin/ (基本命令)
└── sbin/ (系统命令)

💡 所有路径都从 / 开始，没有 C:、D: 盘符

/ - 根目录

  • Linux 文件系统的最顶层
  • 所有文件和目录都在根目录下
  • 类似 Windows 的"我的电脑"，但更统一
  • Windows：C:\、D:\、E:\ (多个根)
  • Linux：只有一个根 /
  • 其他磁盘通过挂载到某个目录

🏠 /home - 用户的家

  • 每个用户都有自己的主目录
  • 路径：/home/用户名
  • 类似 Windows 的 C:\Users\用户名
  • 用户的个人文件都存放在这里
  • 用 ~ 符号代表当前用户的主目录
  • 例如：~/Documents = /home/student/Documents
$ cd ~ # 回到主目录
$ cd ~/Desktop # 进入桌面目录

⚙️ /etc - 配置文件的宝库

  • 存放系统配置文件
  • 网络配置、用户信息、服务设置
  • 重要文件：
    • /etc/passwd - 用户账户信息
    • /etc/hosts - 主机名映射
    • /etc/fstab - 文件系统挂载
  • 修改这些文件通常需要管理员权限

📦 /usr - 程序的居所

  • Unix System Resources 的缩写
  • 存放用户程序和数据
  • 类似 Windows 的 C:\Program Files
  • 子目录：
    • /usr/bin - 用户命令
    • /usr/lib - 程序库
    • /usr/share - 共享数据
    • /usr/local - 本地安装的软件

📊 /var - 变化的数据

  • Variable 的缩写，存放经常变化的数据
  • 重要子目录：
    • /var/log - 系统日志（排查问题必看）
    • /var/cache - 应用缓存
    • /var/tmp - 临时文件（不会自动清理）
    • /var/www - 网站文件（Web 服务器）
  • 💡 系统出问题？先看 /var/log

📂 其他重要目录

/tmp

临时文件

重启后会被清空

/dev

设备文件

硬盘、USB、终端

/bin

基本命令

ls、cp、mv 等

/sbin

系统命令

需要管理员权限

🛤️ 路径的概念

如何描述文件的位置

📍 绝对路径 vs 相对路径

绝对路径（从根目录开始）
/home/student/Documents/report.txt

✅ 无论在哪里，路径都一样

相对路径（从当前位置开始）
Documents/report.txt

✅ 更简洁，但取决于当前位置

💡 绝对路径以 / 开头，相对路径不以 / 开头

🔣 特殊路径符号

  • . (点) - 当前目录
  • .. (两个点) - 上级目录
  • ~ (波浪号) - 当前用户主目录
  • / (斜杠) - 根目录
  • - (减号) - 上一个工作目录
$ cd .. # 返回上级目录
$ cd ~ # 回到主目录
$ cd - # 回到上一个目录
$ cd ../.. # 返回上两级目录

🔄 Windows vs Linux 文件系统

Windows

  • 多个根：C:\、D:\、E:\
  • 路径分隔符：\
  • 不区分大小写
  • 隐藏文件：属性设置
  • 示例：C:\Users\Name\file.txt

Linux

  • 单一根：/
  • 路径分隔符：/
  • 区分大小写
  • 隐藏文件：以 . 开头
  • 示例：/home/name/file.txt

💡 Linux 中 File.txt 和 file.txt 是两个不同的文件！

⌨️ Linux 基础命令

开始动手操作

💡 所有命令都在 Shell 中执行

📍 pwd - 我在哪里？

pwd (Print Working Directory)
显示当前工作目录的完整路径
student@ubuntu:~$
pwd
/home/student
student@ubuntu:~$
cd /etc
student@ubuntu:/etc$
pwd
/etc

💡 迷路了？用 pwd 看看自己在哪

🚶 cd - 切换目录

cd (Change Directory)
在目录之间移动
cd /home/student # 绝对路径
cd Documents # 相对路径
cd .. # 返回上级
cd ~ # 回到主目录
cd - # 回到上一个目录
cd # 回到主目录（无参数）

💡 cd 是最常用的命令之一

👀 ls - 查看文件列表

ls (List)
列出目录中的文件和子目录
ls # 基本列表
ls -l # 详细信息（权限、大小、时间）
ls -a # 显示隐藏文件（以 . 开头）
ls -lh # 人类可读的文件大小（KB、MB）
ls -R # 递归显示子目录
ls -lt # 按修改时间排序
ls -lah # 组合使用多个参数

🔍 理解 ls -l 的输出

drwxr-xr-x 2 student student 4096 Mar 9 10:30 Documents
-rw-r--r-- 1 student student 1024 Mar 8 15:20 file.txt
  • d = 目录，- = 普通文件
  • rwxr-xr-x = 权限（读、写、执行）
  • 2 = 链接数
  • student student = 所有者和组
  • 4096 = 文件大小（字节）
  • Mar 9 10:30 = 修改时间
  • Documents = 文件名

✨ touch - 创建文件

touch
创建空文件或更新文件时间戳
touch file.txt # 创建单个文件
touch file1 file2 file3 # 创建多个文件
touch report.{txt,pdf} # 创建 report.txt 和 report.pdf

💡 如果文件已存在，touch 会更新其修改时间

📁 mkdir - 创建目录

mkdir (Make Directory)
创建新目录
mkdir mydir # 创建单个目录
mkdir dir1 dir2 dir3 # 创建多个目录
mkdir -p project/src/main # 创建多级目录

💡 -p 参数很重要：自动创建父目录

# 没有 -p 会报错（如果 project 不存在）
$ mkdir project/src/main
mkdir: cannot create directory 'project/src/main': No such file or directory

# 使用 -p 自动创建所有层级
$ mkdir -p project/src/main # ✅ 成功

📋 cp - 复制文件

cp (Copy)
复制文件或目录
cp file1.txt file2.txt # 复制文件
cp file.txt /home/student/ # 复制到指定目录
cp -r dir1 dir2 # 复制目录（递归）
cp -i file1 file2 # 交互式（覆盖前询问）
cp -p file1 file2 # 保留属性（时间、权限）

💡 复制目录必须加 -r 参数

🚚 mv - 移动和重命名

mv (Move)
移动文件或重命名
mv old.txt new.txt # 重命名文件
mv file.txt /tmp/ # 移动文件
mv file.txt /tmp/new.txt # 移动并重命名
mv dir1 dir2 # 重命名目录
mv -i file1 file2 # 交互式（覆盖前询问）

💡 mv 不需要 -r 参数，可以直接移动目录

🗑️ rm - 删除文件

rm (Remove)
删除文件或目录
rm file.txt # 删除文件
rm file1 file2 file3 # 删除多个文件
rm -r dir # 删除目录（递归）
rm -i file.txt # 交互式（删除前询问）
rm -f file.txt # 强制删除（不询问）
rm -rf dir # 强制递归删除

⚠️ 警告：Linux 没有回收站，删除即永久！

⚠️ 永远不要运行：rm -rf /（会删除整个系统）

📄 cat - 查看文件内容

cat (Concatenate)
显示文件内容
cat file.txt # 显示文件内容
cat file1 file2 # 显示多个文件
cat -n file.txt # 显示行号

💡 适合查看小文件，大文件用 less

📖 less - 分页查看

less
分页显示文件内容，适合大文件
less file.txt

常用操作：

  • 空格 - 下一页
  • b - 上一页
  • /关键词 - 搜索
  • n - 下一个搜索结果
  • q - 退出

🔝🔚 head 和 tail

head - 查看开头

head file.txt # 前 10 行
head -n 20 file.txt # 前 20 行
head -n 5 file.txt # 前 5 行

tail - 查看结尾

tail file.txt # 后 10 行
tail -n 20 file.txt # 后 20 行
tail -f log.txt # 实时查看日志

💡 tail -f 非常适合查看实时日志

❓ 获取帮助

  • man 命令 - 查看命令手册（最详细）
  • 命令 --help - 快速查看用法
  • info 命令 - 查看详细文档
  • whatis 命令 - 简短描述
$ man ls # 查看 ls 的完整手册
$ ls --help # 快速查看 ls 的用法
$ whatis ls # ls 的简短描述

💡 man 是 manual（手册）的缩写

❗ 常见问题和解决

  • 权限被拒绝：使用 sudo 命令（管理员权限）
  • 命令未找到：检查拼写，或软件未安装
  • 目录不存在：检查路径是否正确
  • 文件已存在：使用 -i 参数交互式操作
  • 终端卡住：按 Ctrl+C 中断当前命令
  • 误删文件：Linux 无回收站，养成备份习惯
