📝 文本处理

Linux 最强大的功能之一

在 Linux 中，一切配置都是文本文件

掌握文本处理，就掌握了系统的核心

1️⃣ grep - 文本搜索

grep = Global Regular Expression Print

就像 Ctrl+F 搜索，但更强大！

$ grep "error" log.txt
Error: Connection failed
Error: File not found
$ grep -i "ERROR" log.txt
# -i 忽略大小写（error、Error、ERROR 都能找到）
  • grep "pattern" file - 在文件中找关键词
  • grep -i - 不区分大小写（更宽松）
  • grep -r - 在整个文件夹里找
  • grep -n - 告诉我在第几行
  • grep -v - 找不包含这个词的行

grep 实用示例

$ grep -r "TODO" .
# 在当前目录递归搜索 TODO
$ grep -n "function" script.js
# 显示包含 function 的行号
$ grep -v "^#" config.txt
# 过滤掉注释行（以 # 开头）
$ grep -c "error" log.txt
# 统计匹配的行数

🔀 管道与重定向

连接命令，组合威力

管道（|）让命令的输出成为另一个命令的输入

💡 类比：就像工厂流水线，上一道工序的产品
直接传给下一道工序，不用中间存储

这是 Unix 哲学的核心：小工具组合完成大任务

管道符 |

管道将一个命令的输出传递给另一个命令

想象成：命令 A 的结果 → 直接喂给 → 命令 B

ls -l
→
grep ".txt"
→
wc -l
$ ls -l | grep ".txt" | wc -l
15
# 统计当前目录有多少个 .txt 文件
# 第一步：列出所有文件
# 第二步：只保留 .txt 的
# 第三步：数有多少行

重定向符号

把命令的输出保存到文件，而不是显示在屏幕上

  • > - 输出重定向（覆盖）← 小心！会覆盖原文件
  • >> - 输出重定向（追加）← 安全，追加到末尾
  • < - 输入重定向（从文件读取）
  • 2> - 错误输出重定向（只保存错误信息）
  • &> - 所有输出重定向（正常+错误都保存）
$ echo "Hello" > file.txt
# 写入文件（如果文件存在会被清空！）
$ echo "World" >> file.txt
# 追加到文件末尾（不会删除原内容）
$ command 2> error.log
# 只把错误信息保存到日志

强大的组合

$ cat access.log | grep "404" | wc -l > 404_count.txt
# 统计 404 错误数量并保存
$ ps aux | grep "python" | grep -v "grep"
# 查找 python 进程（排除 grep 自己）
$ ls -l | sort -k5 -n | tail -10
# 找出最大的 10 个文件

⚙️ 进程管理

掌控系统运行

进程是运行中的程序

学会管理进程，就能控制系统资源

🔄 什么是进程？

💡 简单理解：程序是菜谱，进程是正在做的菜

  • 进程是运行中的程序实例
  • 每个进程有唯一的 PID（进程 ID）
    就像每个人的身份证号
  • 进程有父子关系
    父进程启动子进程，就像家族树
  • 进程占用 CPU、内存等资源
    运行程序需要消耗系统资源
程序（硬盘上的文件）→ 双击运行 → 进程（内存中运行）
例如：chrome.exe（程序）→ 打开浏览器 → Chrome 进程

ps - 查看进程

ps = Process Status（进程状态）

就像 Windows 的任务管理器！

$ ps
PID TTY TIME CMD
1234 pts/0 00:00:00 bash
# 只显示当前终端的进程
$ ps aux
# 显示所有进程的详细信息（最常用！）
# a = 所有用户 u = 详细信息 x = 包括后台进程
  • ps - 当前终端的进程（很少用）
  • ps aux - 所有进程（推荐！）
  • ps -ef - 完整格式显示（另一种风格）

top - 实时监控

top 实时显示系统资源使用情况

就像汽车仪表盘，实时显示系统状态！

top - 10:30:15 up 5 days, 2:15, 3 users
# 当前时间 运行了5天 3个用户登录
Tasks: 245 total, 1 running, 244 sleeping
# 总共245个进程，1个在运行，244个在睡眠
%Cpu(s): 12.5 us, 3.2 sy, 0.0 ni, 84.3 id
# CPU使用率：12.5%用户程序，84.3%空闲
MiB Mem: 16384 total, 8192 free, 6144 used
# 内存：总共16G，空闲8G，使用6G
PID USER %CPU %MEM COMMAND
1234 user 25.0 5.2 python app.py
# 这个进程占用25%的CPU，5.2%的内存
  • 按 q 退出
  • 按 M 按内存排序（找内存大户）
  • 按 P 按 CPU 排序（找CPU大户）

kill - 终止进程

kill 发送信号给进程

就像关闭程序，但更强大！

$ kill 1234
# 礼貌地请求进程退出（进程可以拒绝）
$ kill -9 1234
# 强制杀死进程（进程无法拒绝，立即终止）
# ⚠️ 类似"强制关机"，可能丢失数据
$ killall firefox
# 杀死所有名为 firefox 的进程
  • kill PID - 正常终止（推荐）
  • kill -9 PID - 强制终止（最后手段）
  • killall name - 按名称终止所有

⚠️ 使用 kill -9 前先试试普通 kill，给程序保存数据的机会

后台运行进程

让程序在后台运行，你可以继续做其他事

$ command &
# 在后台运行命令（加个 & 符号）
# 就像把程序最小化，它还在运行但不占用终端
$ jobs
[1]+ Running python script.py &
# 查看后台任务列表
$ fg %1
# 将后台任务调到前台（fg = foreground）
$ Ctrl+Z # 暂停当前任务
$ bg
# 让暂停的任务在后台继续运行（bg = background）

💡 实用场景：运行一个需要很长时间的程序，同时你还想用终端做其他事

🌐 网络命令

连接世界

ping - 测试连接

ping 测试网络连通性

就像喊一声"喂！"，看对方能不能听到并回应

$ ping google.com
PING google.com (142.250.185.46): 56 data bytes
64 bytes from 142.250.185.46: icmp_seq=0 time=15.2 ms
# time=15.2ms 表示往返时间，越小越好
64 bytes from 142.250.185.46: icmp_seq=1 time=14.8 ms
# 会一直 ping，按 Ctrl+C 停止
$ ping -c 4 google.com
# 只 ping 4 次就停止（-c = count）
  • 测试网络是否连通（能不能访问）
  • 查看网络延迟（速度快不快）
  • time < 50ms 很快，> 200ms 就慢了

curl - 网络请求

curl 发送 HTTP 请求

就像用浏览器访问网页，但在命令行里！

$ curl https://api.github.com
# 获取网页内容（会显示在屏幕上）
$ curl -o page.html https://example.com
# 保存到文件 page.html（-o = output）
$ curl -I https://google.com
# 只看响应头，不下载内容（-I = head）
# 可以看到服务器类型、内容大小等信息
  • 测试 API 接口（程序员常用）
  • 下载文件
  • 调试网络问题

💡 curl 很强大，可以模拟浏览器做各种操作

wget - 下载文件

wget 下载网络文件

专业的下载工具，支持断点续传！

$ wget https://example.com/file.zip
# 下载文件（会显示进度条）
$ wget -c https://example.com/large.iso
# 断点续传（-c = continue）
# 如果下载中断，再次运行会从断点继续
$ wget -r -np https://example.com/docs/
# 递归下载整个目录（-r = recursive）
# 可以下载整个网站！

💡 wget vs curl：wget 专注下载，curl 更灵活
下载大文件用 wget，测试 API 用 curl

🔍 find - 查找文件

find 强大的文件搜索工具

就像 Windows 的搜索，但功能强大 100 倍！

$ find . -name "*.txt"
# 在当前目录（.）查找所有 .txt 文件
$ find /home -user student
# 查找 student 用户的所有文件
$ find . -type f -size +100M
# 查找大于 100MB 的文件（找出占空间的大文件）
$ find . -mtime -7
# 查找 7 天内修改的文件（找最近改过的）

💡 find 可以按名称、大小、时间、权限等各种条件查找

  • 查找 firefox 进程的 PID
  • （模拟）终止 PID 为 1234 的进程

答案：

# 1. 查看所有进程

# 3. 查找 firefox 的 PID
$ ps aux | grep firefox
# 或者
$ pgrep firefox
# 4. 终止进程
$ kill 1234
# 如果不响应，使用 kill -9 1234

📝 综合练习

挑战任务：

组合使用多个命令完成以下任务：

  • 找出当前目录最大的 5 个文件
  • 统计日志文件中 ERROR 出现的次数并保存
  • 找出占用 CPU 最多的进程并显示详细信息

答案：

# 1. 最大的 5 个文件
$ ls -lh | sort -k5 -h | tail -5
# 或者使用 du
$ du -ah . | sort -h | tail -5

$ grep -c "ERROR" app.log > error_count.txt
# 或者
$ grep "ERROR" app.log | wc -l > error_count.txt
# 3. CPU 占用最高的进程
$ ps aux --sort=-%cpu | head -2
# head -2 显示标题和第一个进程
