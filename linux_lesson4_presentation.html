🤔 为什么需要用户管理？

💡 Linux 是多用户多任务操作系统

  • 安全性 - 隔离不同用户的数据和权限
  • 隐私保护 - 每个用户有独立的工作空间
  • 资源管理 - 控制用户对系统资源的访问
  • 责任追踪 - 记录谁做了什么操作

就像公司里每个员工有自己的工位和权限

👤 什么是用户？

用户是系统访问控制的基本单位

👨‍💼 👩‍💻 🤖
  • root 用户 - 超级管理员（UID=0）
    拥有所有权限，可以做任何事
  • 系统用户 - 运行系统服务（UID 1-999）
    如 nginx、mysql 等服务的专用账户
  • 普通用户 - 日常使用（UID ≥1000）
    我们平时登录使用的账户

每个用户有唯一的 用户名 和 UID（用户ID）

👥 什么是组？

组是用户的集合，用于简化权限管理

💡 类比：组就像公司的部门

  • 主组（Primary Group）
    用户创建文件时的默认组
  • 附加组（Secondary Group）
    用户可以加入的其他组
# 例如：创建 developers 组
# 将 alice、bob、charlie 加入该组
# 给 developers 组分配项目目录权限
# 这样三个人都能访问项目文件

用户和组的关系

一个用户可以属于多个组

👤 用户 john
├─ 主组: john (GID 1000)
└─ 附加组:
├─ developers (GID 1001)
├─ docker (GID 999)
└─ sudo (GID 27)

john 创建的文件默认属于 john 组
但他可以访问 developers、docker、sudo 组的资源

📁 用户和组配置文件

了解系统如何存储用户信息

  • /etc/passwd - 用户账户信息
  • /etc/shadow - 用户密码信息（加密）
  • /etc/group - 组信息
  • /etc/gshadow - 组密码信息

这些文件是 Linux 用户管理的核心

/etc/passwd 文件

/etc/passwd 存储所有用户账户信息

所有用户可读，只有 root 可以修改

$ cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
john:x:1000:1000:John Smith:/home/john:/bin/bash
nginx:x:33:33:Nginx:/var/www:/usr/sbin/nologin

格式： 用户名:密码:UID:GID:描述:家目录:Shell

  • x 表示密码存储在 /etc/shadow
  • UID 0 是 root，≥1000 是普通用户
  • /usr/sbin/nologin 表示不能登录

/etc/shadow 文件

/etc/shadow 存储加密的密码信息

🔒 只有 root 可以读取，安全性更高

# sudo cat /etc/shadow
john:$6$xyz...:18500:0:99999:7:::
# $6$ 表示使用 SHA-512 加密

格式： 用户名:加密密码:最后修改:最小间隔:最大间隔:警告期:...

  • 密码使用强加密算法
  • 包含密码过期策略
  • 提高系统安全性

/etc/group 文件

/etc/group 存储所有组信息

$ cat /etc/group
root:x:0:
developers:x:1001:alice,bob,charlie
docker:x:999:john,alice

格式： 组名:密码:GID:组成员列表

  • 组成员用逗号分隔
  • 用户的主组不会显示在成员列表中

🔧 用户管理命令

创建、修改、删除用户

useradd

创建新用户

usermod

修改用户属性

userdel

删除用户

passwd

设置密码

useradd - 创建用户

useradd 创建新用户账户

# useradd john
# 创建用户 john（最简单）
# useradd -m alice
# 创建用户并创建家目录（推荐）
# useradd -m -s /bin/bash bob
# 创建用户并指定 Shell
# useradd -m -G developers,docker charlie
# 创建用户并加入多个组
  • -m 创建家目录
  • -s 指定登录 Shell
  • -G 指定附加组

passwd - 设置密码

passwd 设置或修改用户密码

创建用户后必须设置密码才能登录

# passwd john
New password: [输入密码]
Retype new password: [再次输入]
passwd: password updated successfully
$ passwd
# 普通用户修改自己的密码

🔒 密码建议：至少8位，包含大小写字母、数字和特殊字符

usermod - 修改用户

usermod 修改现有用户的属性

# usermod -aG docker john
# 将 john 添加到 docker 组（最常用！）
# usermod -l newname oldname
# 修改用户名
# usermod -s /bin/zsh alice
# 修改用户的 Shell
# usermod -L bob
# 锁定用户账户
# usermod -U bob
# 解锁用户账户

⚠️ 注意：-aG 是添加组，-G 会覆盖所有附加组

userdel - 删除用户

userdel 删除用户账户

# userdel john
# 删除用户（保留家目录）
# userdel -r alice
# 删除用户及其家目录（常用）
# userdel -rf bob
# 强制删除（即使用户已登录）

⚠️ 警告：删除前请备份重要数据，-r 会永久删除家目录

# 删除前检查
# who | grep john
# 查看用户是否登录
# ps -u john
# 查看用户的进程

👥 组管理命令

创建、修改、删除组

groupadd

创建新组

groupmod

修改组属性

groupdel

删除组

gpasswd

管理组成员

groupadd - 创建组

groupadd 创建新的用户组

# groupadd developers
# 创建组（系统自动分配 GID）
# groupadd -g 2000 testers
# 创建组并指定 GID
# grep developers /etc/group
developers:x:1001:
# 验证组创建成功

groupmod / groupdel

groupmod 修改组，groupdel 删除组

# 修改组名
# groupmod -n newgroup oldgroup
# 修改 GID
# groupmod -g 3000 developers
# 删除组
# groupdel developers

⚠️ 不能删除用户的主组，必须先删除或修改用户

🔍 查看用户信息

id 命令显示用户和组信息

$ id
uid=1000(john) gid=1000(john) groups=1000(john),27(sudo),999(docker)
# 显示当前用户的 UID、GID 和所有组
$ id alice
uid=1001(alice) gid=1001(alice) groups=1001(alice),1002(developers)
# 查看指定用户
$ id -u
1000
# 只显示 UID

💡 用途：快速查看用户所属的所有组，排查权限问题

su - 切换用户

su（Switch User）切换到其他用户

$ su -
Password: [输入 root 密码]
# [现在是 root 用户]
$ su - alice
# 切换到 alice 用户
$ exit
# 退出当前用户，返回之前的用户
  • su - 切换但不改变环境变量
  • su - - 完整切换（推荐）
  • 需要目标用户的密码

sudo - 临时提权

sudo（SuperUser DO）以 root 权限执行命令

💡 更安全的方式，不需要知道 root 密码

$ sudo apt install nginx
[sudo] password for john: [输入自己的密码]
# 以 root 权限安装软件
$ sudo systemctl restart nginx
# 重启服务
$ sudo -i
# 切换到 root Shell
  • 只需要输入自己的密码
  • 有操作日志记录
  • 可以精细控制权限

sudo vs su

sudo

  • 执行单个命令
  • 输入自己的密码
  • 有操作日志
  • 可以精细控制权限
  • 更安全（推荐）

su

  • 完全切换用户
  • 需要目标用户密码
  • 无详细日志
  • 获得完整权限
  • 适合长时间管理

🔒 建议：日常使用 sudo，避免直接使用 root 账户

查看登录用户

who 和 w 命令

$ who
john pts/0 2024-03-30 10:30 (192.168.1.100)
alice pts/1 2024-03-30 11:00 (192.168.1.101)
# 显示当前登录的用户
$ w
# 显示更详细的信息（用户在做什么）
$ whoami
john
# 显示当前用户名

🎯 实战演练

创建一个开发团队的用户环境

# 1. 创建 developers 组
# groupadd developers
# 2. 创建三个开发者账户
# useradd -m -G developers -s /bin/bash alice
# useradd -m -G developers -s /bin/bash bob
# useradd -m -G developers -s /bin/bash charlie
# 3. 设置密码
# passwd alice
# passwd bob
# passwd charlie
# 4. 创建共享项目目录
# mkdir /home/projects
# chgrp developers /home/projects
# chmod 770 /home/projects
# 现在 developers 组的成员都能访问项目目录

🔐 密码策略管理

chage 命令管理密码过期信息

# chage -l john
# 查看用户的密码过期信息
# chage -M 90 john
# 设置密码90天后过期
# chage -d 0 john
# 强制用户下次登录时修改密码
# chage -E 2024-12-31 john
# 设置账户过期日期

💡 用于增强系统安全性，定期更换密码

💼 常见应用场景

  • Web 服务器
    创建 www-data 用户运行 nginx/apache
  • 数据库服务器
    创建 mysql 用户运行数据库服务
  • 开发团队
    创建 developers 组，共享项目目录
  • 容器环境
    将用户加入 docker 组，无需 sudo 运行容器
  • 系统管理
    将用户加入 sudo 组，授予管理权限

🔒 安全最佳实践

  • 不要直接使用 root 账户登录
  • 使用 sudo 而不是 su
  • 设置强密码策略（长度、复杂度）
  • 定期更换密码
  • 及时删除不用的账户
  • 使用 SSH 密钥认证代替密码
  • 限制 sudo 权限，只授予必要的命令
  • 定期审查用户和组

🛡️ 安全是系统管理的首要任务
